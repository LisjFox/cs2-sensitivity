## 一、为什么做这个项目？

### 1.1 痛点分析

《反恐精英 2》(CS2) 是一款对鼠标精度要求极高的竞技射击游戏。职业选手的灵敏度往往精确到小数点后四位，而普通玩家普遍存在以下问题：

- **不知道怎么调** — 游戏内有 "灵敏度" 和 "DPI" 两个参数，两者组合成 eDPI，多数玩家不理解它们的关系
- **没有参考标准** — 不知道什么样的灵敏度适合自己，盲目模仿职业选手反而得不偿失
- **调了不敢改** — 修改配置文件有风险，改坏了不知道如何恢复
- **缺乏客观数据** — 判断灵敏度是否合适，全凭"手感"，缺少量化指标

### 1.2 解决方案

本项目将 AI 助手与自动化工具结合，提供一套**完整闭环**的灵敏度优化体验：

1. AI 引导用户完成信息采集（角色定位、鼠标参数、主观感受）
2. 自动读取本机 CS2 配置文件，获取当前设置
3. 基于游戏角色 + 职业选手基准 + 鼠标物理条件，计算推荐方案
4. 一键写入配置，自动备份，零风险

## 二、核心功能

### 2.1 自动配置读取（`read_cs2_config.py`）

- 自动扫描 Steam userdata 目录和 Documents 目录
- 支持多 Steam 安装路径（C 盘 / D 盘等）
- 解析 KeyValues 格式的 vcfg 配置文件
- 输出：灵敏度、缩放灵敏度比例、分辨率等

### 2.2 智能计算引擎（`calc_sensitivity.py`）

输入当前设置 + 用户信息，计算并推荐：

| 指标 | 说明 |
|------|------|
| **eDPI** | 鼠标 DPI × 游戏内灵敏度，统一量化标准 |
| **cm/360°** | 鼠标移动多少厘米能让视角转一圈 |
| **灵敏度区间** | 极低敏 / 低敏 / 中敏 / 高敏 四档定性 |
| **角色推荐** | 突破手、狙击手、自由人等各有推荐区间 |
| **鼠标垫匹配** | 检查鼠标垫宽度是否能完成完整转身 |

内置职业选手基准数据库（NiKo、s1mple、ZywOo、woxic 等）。

### 2.3 一键应用（`apply_sensitivity.py`）

- 自动定位 CS2 配置文件
- 修改前**自动备份**原文件到 `.bak`
- 精确保留文件格式，不破坏其他配置项

### 2.4 战绩辅助（`fetch_cs2_stats.py`，可选）

- 通过 Steam Web API 获取 K/D、爆头率、场均伤害等
- 自动从本地 Steam 读取用户 Steam ID
- 数据辅助 AI 做更精准的判断

### 3.2 技术选型

| 层次 | 技术 | 说明 |
|------|------|------|
| AI 对话框架 | OpenClaw Skill | 基于 SKILL.md 定义的工具调用协议 |

### 3.3 工作流程

```
用户触发话题
    ↓
AI 读取 SKILL.md 了解能力范围
    ↓
AI 运行 read_cs2_config.py → 获取当前配置
    ↓
AI 向用户提问：鼠标 DPI？角色定位？鼠标垫尺寸？主观感受？
    ↓
AI 运行 calc_sensitivity.py → 生成诊断报告
    ↓

    ↓（用户同意）
AI 运行 apply_sensitivity.py → 自动备份 + 写入
    ↓
提示：重启 CS2 生效，备份文件位置
```

## 四、开发过程中的困难与解决

### 困难 1：CS2 配置文件格式不统一

**问题：** CS2 更新后配置文件可能放在不同路径（Steam userdata / Documents / 不同盘符），且 KeyValues 格式解析容易出错。

**解决：**
- 遍历多个可能的 Steam 安装路径
- 同时检查较新的 Documents 目录路径
- 使用正则表达式精确匹配 `"key" "value"` 结构，忽略花括号嵌套
- 非关键字段解析失败不中断整体流程

### 困难 2：AI 工具调用的上下文理解

**问题：** AI 需要在多轮对话中保持上下文（用户说了什么、上一步计算结果是什么），而工具调用是独立的 Python 进程，没有状态。

**解决：**
- 定义严格的 JSON 输入/输出协议，工具之间通过标准格式通信
- 所有计算结果由 AI 在对话中呈现和解释，工具只做数据计算
- SKILL.md 中定义了完整的工作流程，指导 AI 按步骤执行

### 困难 3：安全风险控制

**问题：** 自动修改配置文件有风险，如果脚本有 bug 可能破坏玩家的 CS2 设置。

**解决：**
- 每次写入前**强制自动备份**原文件（`.bak` 后缀）
- 备份文件已有则不覆盖，防止误操作
- `apply_sensitivity.py` 使用正则定位精确替换灵敏度数值，不触碰其他配置项
- AI 在修改前再次向用户确认，人类始终在决策链中

### 困难 4：中文编码（Windows 环境）

**问题：** Windows 终端默认编码与 Python 脚本编码不一致，导致中文输出乱码。

**解决：**
- 所有 JSON 输出使用 `ensure_ascii=False` 参数
- 文件读取使用 `encoding="utf-8", errors="ignore"` 增加容错
- 最终方案是转为英文输出 + AI 在对话层做中文呈现

## 六、对 AI 的思考与感悟

### 6.1 AI 不是"万能答案机"

做这个项目之前，我以为 AI 开发就是写一个 prompt 扔给大模型。实际做下来才发现：**工具是 AI 能力的放大器，但工具本身需要精心设计。**

AI 再强，也需要有人为它搭好"脚手架"——定义好数据结构、设计好工作流程、写好容错逻辑。AI 是大脑，工具是手脚，两者缺一不可。

### 6.2 对话式交互的未来

传统工具（比如 CS2 灵敏度计算器）通常是网页或桌面应用，用户需要自己填参数、看文档、摸索使用方式。而基于 AI Agent 的方案让用户用自然语言就能完成复杂任务：

> "我打狙击的，800 DPI，鼠标垫 40cm，帮我调一下灵敏度。"

AI 理解这句话后，自动读取配置、计算、写入。**交互方式的简化，降低了专业知识的获取门槛。**

### 6.3 安全与信任

让 AI 自动修改系统配置文件，本质上是把信任交给了代码。为了保证这种信任是合理的，我们在设计中坚持了一个原则：**AI 建议，人类决策。**

- AI 计算推荐值，但展示完整的推理过程（eDPI、cm/360°、角色推荐区间）
- 最终写入前必须经过用户确认
- 所有修改都是可逆的（自动备份）

这点对于 AI 落地的任何一个场景都适用——让 AI 增强人，而不是替代人。

### 6.4 一个小感悟

最让我意外的是：整个项目中**最复杂的部分不是 AI 的对话逻辑，而是「读 CS2 配置」和「算灵敏度」这两个看似简单的 Python 脚本。** 文件路径、编码格式、正则表达式、API 错误处理……这些"脏活"才是一个项目真正落地时的难点。

AI 是锦上添花，扎实的工程能力才是地基。
